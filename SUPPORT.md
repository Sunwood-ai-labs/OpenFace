# Support

Use the following path for help:

- Search existing GitHub issues for the symptom.
- Use the bug report form for reproducible defects.
- Use the feature request form for product proposals.
- Use private vulnerability reporting for security-sensitive problems.

For local diagnostics, include sanitized output from `docker compose ps` and the relevant service logs. Never attach `.env`, access tokens, generated certificates, databases, or clone URLs containing credentials.
